2. Platform RTMP checklist

▶ YouTube LiveRTMP server:
rtmp://a.rtmp.youtube.com/live2

Create a stream in YouTube Studio and copy your stream key into OBS.

f Facebook LiveRTMPS server:
rtmps://live-api-s.facebook.com:443/rtmp/

Use Facebook Live Producer; paste its key in OBS.

🎵 TikTok Live

TikTok must enable Live/RTMP access on the account. Copy the URL/key from TikTok Live Center or TikTok Studio.

📸 Instagram Live

Use Instagram Live Producer if your account has it. Instagram supplies a temporary RTMP URL and key.

Important limitation: browsers cannot send RTMP directly to social media from a static GitHub Pages site. Use free OBS Studio or a free relay. Do not paste stream keys into ClassDeck.

4. Enterprise production checklist

  1. Consent & privacy: announce if the class will be public; hide minors' names/faces unless you have permission.
  2. Bandwidth: direct multistreaming needs upload bandwidth for every platform. A relay uses one upload but may limit free destinations.
  3. Audio: in OBS, set mic noise suppression/noise gate; test with a private/unlisted stream first.
  4. Vertical platforms: TikTok/Instagram prefer 9:16. Use a second OBS scene/canvas or crop the ClassDeck output.
  5. Backup: start ClassDeck local recording (⏺ Rec) so you still have a copy if a platform disconnects.
  6. Engagement: keep ClassDeck chat/polls/quiz for enrolled students; read selected questions aloud for the public audience.
Recommended free stack: ClassDeck → OBS Window Capture → one platform directly. For multistream: OBS Multiple RTMP Outputs plugin or a free relay account if available in your region.
